Как скопировать на свой компьютер репозиторий из github.com, что должно быть сделано предварительно?
	    
Установить Git на свой компьютер, если он еще не установлен.
Создать пустую папку на своем компьютере, в которую будет скопирован репозиторий.
Скопировать URL-адрес репозитория с GitHub, для этого нужно зайти на страницу репозитория на GitHub и нажать на зеленую кнопку "Code". В открывшемся окне выбрать протокол, по которому будет происходить копирование репозитория        (HTTPS или SSH), скопировать URL-адрес.
Открыть командную строку или Git Bash и перейти в папку, которую вы создали в пункте 2.
Выполнить команду git clone с указанием скопированного URL-адреса репозитория:
git clone <URL-адрес репозитория>
Git выполнит клонирование репозитория в указанную папку на компьютере.



Как создать git репозиторий из локального каталога на своем компьютере. Как его отправить на github.com?

Создать локальный git репозиторий. Для этого нужно открыть командную строку или Git Bash, перейти в нужный каталог с помощью команды cd, и выполнить команду:
git init
Добавить файлы в репозиторий. Для этого нужно создать файлы в каталоге репозитория или скопировать туда уже существующие. После этого нужно выполнить команду:
git add <название файла>   # добавление конкретного файла в репозиторий
git add .                  # добавление всех файлов в репозиторий
Сделать коммит изменений. Для этого нужно выполнить команду:
git commit -m "комментарий к коммиту"
Создать репозиторий на GitHub. Для этого нужно зайти на страницу GitHub, нажать на кнопку "New" и заполнить необходимые поля.
Связать локальный репозиторий с репозиторием на GitHub. Для этого нужно выполнить команду:
git remote add origin <URL-адрес репозитория на GitHub>
Отправить локальные изменения на GitHub. Для этого нужно выполнить команду:
git push -u origin master(main)
Где origin - это имя удаленного репозитория, а master(main) - это имя ветки, которая будет отправлена на сервер. Если в репозитории на GitHub уже есть какие-то файлы, то можно вместо master(main) указать название нужной ветки. После выполнения этой команды изменения будут отправлены на сервер и сохранены в удаленном репозитории.



Как получить на локальный компьютер изменения с github.com

Для получения изменений с удаленного репозитория на локальный компьютер можно использовать команду git pull. Эта команда загружает все изменения, которые были внесены в удаленный репозиторий и объединяет их с локальной веткой.
Чтобы выполнить git pull, нужно перейти в локальный репозиторий с помощью команды cd и выполнить следующие команды:
git pull <remote_name> <branch_name>
Здесь <remote_name> - это имя удаленного репозитория, откуда нужно получить изменения, а <branch_name> - это имя ветки, которую нужно обновить.
Например, если удаленный репозиторий называется origin, а ветка, которую нужно обновить, называется main, то команда будет выглядеть так:
git pull origin main
Если вы хотите получить изменения из удаленного репозитория в текущую ветку, то можно выполнить команду git pull без указания имени ветки:
git pull origin
В этом случае изменения будут загружены в текущую ветку и автоматически объединены с ней.



Какую информацию нам дают команда :
git status
git log
git remote -v

Команда git status показывает текущее состояние локального репозитория и несохраненные изменения в рабочей директории. Она сообщает о том, какие файлы были изменены, какие файлы добавлены или удалены, и какие изменения ожидают коммита. Это полезная команда для отслеживания состояния локального репозитория.
Команда git log показывает список коммитов в локальном репозитории. Каждый коммит содержит информацию о дате, авторе, комментарии и хэше коммита. Эта команда позволяет просмотреть историю изменений в репозитории и отследить, кто и когда внес изменения в код.
Команда git remote -v показывает список удаленных репозиториев, связанных с локальным репозиторием, а также URL для каждого из них. Она также показывает название, которое было дано удаленному репозиторию (обычно это origin). Эта команда полезна для проверки настроек удаленных репозиториев и для убедительности, что удаленный репозиторий правильно настроен.



До недавнего времени в Git использовалась ветка master как основная ветка в новых репозиториях. Однако, с мая 2021 года в качестве основной ветки по умолчанию используется ветка main.
Поэтому, в зависимости от того, когда был создан репозиторий, в качестве основной ветки может использоваться как master, так и main. Если репозиторий был создан до мая 2021 года, то в качестве основной ветки следует использовать master. Если же репозиторий был создан после мая 2021 года, то нужно использовать main.
В любом случае, для отправки изменений на GitHub можно использовать команду:
git push -u origin <branch-name>
где <branch-name> - название ветки, которую нужно отправить на GitHub (например, master или main). Если ветка на GitHub еще не существует, то она будет создана автоматически.
